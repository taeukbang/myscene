# 맥락 관리 시스템 사용법

## 개요

이 프로젝트는 AI 어시스턴트와의 대화 맥락을 지속적으로 유지하기 위한 문서 시스템을 사용합니다.

## 파일 구조

```
MyScene/
├── PROJECT_CONTEXT.md          # 프로젝트 전체 상태 (주요 파일)
├── CONVERSATION_GUIDE.md       # AI와의 대화 가이드라인
├── HOW_TO_USE_CONTEXT.md      # 이 파일 (사용법)
└── PRD.md                      # 제품 요구사항 문서

~/.claude/projects/.../memory/
└── MEMORY.md                   # AI 자동 메모리 (자동 로드됨)
```

## 각 파일의 역할

### 1. PROJECT_CONTEXT.md (가장 중요)

**목적**: 프로젝트의 현재 상태를 한눈에 파악

**내용**:
- 프로젝트 개요 및 목표
- 기술 스택
- 완료된 작업 ✅
- 진행 중인 작업 🔄
- 예정된 작업 📋
- 발생한 문제와 해결 방법
- 주요 결정 사항
- 다음 단계

**언제 업데이트**:
- 중요한 작업 완료 시
- 새로운 문제 발견 시
- 기술적 결정 변경 시
- 세션 종료 시

### 2. CONVERSATION_GUIDE.md

**목적**: AI 어시스턴트에게 어떻게 대화할지 알려줌

**내용**:
- 사용자 선호도
- 커뮤니케이션 스타일
- 해야 할 것 / 하지 말아야 할 것
- 작업 패턴
- 문제 해결 접근

**언제 업데이트**:
- 새로운 선호도 발견 시
- 반복되는 패턴 발견 시
- AI가 잘못된 접근을 반복할 때

### 3. MEMORY.md (자동 로드)

**목적**: AI가 매 세션마다 자동으로 읽는 메모리

**내용**:
- 핵심 원칙 요약
- 기술적 교훈
- 자주 사용하는 명령어
- 파일 참조 링크

**특징**:
- Claude의 auto memory 시스템에 의해 자동 로드
- 200줄 이후는 잘림 → 간결하게 유지
- 중요한 정보만 기록

## 사용 흐름

### 새 세션 시작 시

1. AI가 자동으로 MEMORY.md 읽음 (시스템이 자동 로드)
2. AI에게 요청: "PROJECT_CONTEXT.md 확인하고 현재 상태 알려줘"
3. 작업 계속 진행

### 작업 진행 중

- AI가 TodoWrite로 작업 추적
- 중요한 결정이나 문제 발생 시 AI가 PROJECT_CONTEXT.md 업데이트

### 세션 종료 시

AI에게 요청: "오늘 작업 내용 PROJECT_CONTEXT.md에 업데이트해줘"

또는 AI가 자동으로:
- 완료된 작업을 ✅ 섹션에 추가
- 새로운 문제/해결 사항 기록
- 다음 단계 업데이트

## 실제 사용 예시

### 예시 1: 새 세션 시작

```
User: "어제 어디까지 했지?"

AI: [PROJECT_CONTEXT.md 읽음]
"Instagram 스크래핑을 포기하고 웹 스크래핑으로 전환 결정했습니다.
다음은 웹 스크래핑 구현입니다. Naver 블로그 스크래핑부터 시작할까요?"
```

### 예시 2: 작업 완료 후

```
User: "스크래퍼 구현 끝났어. 문서 업데이트해줘"

AI: [PROJECT_CONTEXT.md 업데이트]
- ✅ 완료에 "웹 스크래퍼 구현 (Naver 블로그)" 추가
- 🔄 진행 중 업데이트
- 다음 단계 갱신
"업데이트 완료. 다음은 스크래퍼 실행 및 데이터 수집입니다."
```

### 예시 3: 새로운 요구사항

```
User: "카페 말고 레스토랑도 추가하고 싶어"

AI: [PROJECT_CONTEXT.md 업데이트]
- 📋 예정에 "레스토랑 카테고리 추가" 항목 추가
"PROJECT_CONTEXT.md에 추가했습니다. 카페 완성 후 진행하시겠어요?"
```

## 유지보수 팁

### 정기적으로 할 것

1. **매주 리뷰** (PROJECT_CONTEXT.md)
   - 완료된 항목 정리
   - 오래된 정보 제거
   - 다음 단계 명확화

2. **패턴 발견 시** (CONVERSATION_GUIDE.md)
   - AI가 반복하는 실수 기록
   - 효과적인 패턴 추가

3. **교훈 발견 시** (MEMORY.md)
   - 중요한 기술적 교훈 추가
   - 200줄 초과 시 덜 중요한 내용 제거

### 파일이 너무 길어지면

**PROJECT_CONTEXT.md**:
- 완료된 섹션을 `docs/history/` 폴더로 이동
- 현재 진행 중인 것만 남기기

**CONVERSATION_GUIDE.md**:
- 구체적 예시를 별도 파일로 분리
- 핵심 원칙만 남기기

## 장점

1. **세션 간 연속성**
   - 새 대화를 시작해도 맥락 유지
   - "어디까지 했더라?" 고민 불필요

2. **일관된 작업 방식**
   - AI가 사용자 선호도를 기억
   - 반복 설명 불필요

3. **문제 해결 가속화**
   - 과거 문제와 해결책 기록
   - 같은 실수 반복 방지

4. **프로젝트 문서화**
   - 자연스럽게 프로젝트 히스토리 생성
   - 나중에 참고 가능

## 문제 해결

### AI가 맥락을 못 읽을 때

```
User: "PROJECT_CONTEXT.md 읽고 요약해줘"
```

### 정보가 충돌할 때

- PROJECT_CONTEXT.md가 최신 정보 (우선순위 1)
- CONVERSATION_GUIDE.md는 대화 방식 (우선순위 2)
- MEMORY.md는 보조 정보 (우선순위 3)

### 파일 업데이트 빈도

- **높음**: PROJECT_CONTEXT.md (매 세션)
- **중간**: MEMORY.md (주 1-2회)
- **낮음**: CONVERSATION_GUIDE.md (월 1회 또는 필요시)

## 핵심 원칙

> 이 시스템의 목적은 "AI와의 대화를 마치 사람 협업자처럼 연속적으로 만드는 것"입니다.

- 짧고 명확하게 기록
- 최신 정보 유지
- 중요한 것만 기록
- 정기적으로 정리

---

**TIP**: 세션 시작할 때마다 "어디까지 했지?" 대신
"프로젝트 상태 확인해줘"라고 말하면 AI가 PROJECT_CONTEXT.md를 읽고 요약해줍니다.
